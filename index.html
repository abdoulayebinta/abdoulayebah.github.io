Internship / wimmics.inria.fr
Mar 2011 - Dec 2012
Integrate CORESE semantic engine on Android

This internship consists of trying to carry the semantic engine Corese, developed in Java at INRIA, on Android. Corese processes data from the web / semantic web in RDF, RDFS, and supports the SPARQL 1.1 query language. These languages are W3C standards and form the cornerstone of the semantic web.The RDF format is an XML format that can also be embedded in HTML pages to describe "hidden data". RSS feeds are RDF, the data hidden in the pages that contain the "Like" button of Facebook are in RDF, etc. These metadata refer to models, here also XML files present on the web, in RDFS format. Ex: The OpenGraph format of facebook, the format of RSS feeds, etc. Who says data + models says query language. This language is called SPARQL, it's a bit "SQL at the scale of the web". Corese is an example of a semantic database that can handle such data and respond to queries. Particularity of these languages: they allow "inference", or to use less technical words,"reasoning". We can deduce things implicitly from non-explicit data.
